?这是什么？

OOF Manager 是一个轻量的 Windows 桌面工具，帮你自动管理 Microsoft 365 邮箱的自动回复（Out-of-Office）状态。 你只需登录一次，配好工作时间和回复内容，它就会在后台每 5 分钟自动开 / 关你的 OOF —— 再也不用每天手动进 Outlook 设置。

🗓️

按周排程

逐天勾选 + 30 分钟粒度的开关时间。

📋

模板库

预存出差 / 休假等常用文案。

🖥️

静默驻留

最小化到系统托盘后几乎无感。

🔐

本地运行

不依赖云流程，凭据从不落盘。

3安装步骤

图 1 · 安装向导（Inno Setup，一路点 Next 即可）
  1. 双击下载的 OofManagerSetup.exe。
  2. 如果 Windows SmartScreen 弹窗，点击 "More info" → "Run anyway"。
    ⚠️ 关于 SmartScreen：这是因为安装包尚未购买代码签名证书。你可以通过 GitHub Releases 页直接看到源码与构建过程，确保来源可信。
  3. 一路点 Next 直到完成即可——除了"Create a desktop shortcut"（是否创建桌面快捷方式）这一项你可能会想勾上，其它无需修改。
  4. 安装结束后默认会启动程序，开始下一节"首次启动 / 登录"。
💡 安装位置：默认安装到 %LocalAppData%\Programs\OofManager，不需要管理员权限，也不会污染 Program Files。

4首次启动 / 登录

首次打开看到登录界面：

图 2 · 登录页（首次启动）
  1. 点击 🔑 Sign in with Microsoft。
  2. 程序会调用 Connect-ExchangeOnline，弹出 Microsoft 标准登录窗口。
  3. 输入你的工作邮箱 → 走完 MFA / 条件访问流程。
  4. 登录成功后自动跳转到主界面。
🔒 安全说明：登录使用微软现代认证（OAuth 2.0），程序不接触你的密码，也不会把任何凭据写入磁盘。会话令牌由 ExchangeOnlineManagement 模块本身管理。

6配置工作时间排程

这是 OOF Manager 最核心的功能：让程序在你的工作时间外自动开启自动回复。

  1. 打开 🗓️ Work Schedule 卡片右上角的开关，启用排程。
  2. 勾选你的工作日（默认周一至周五；轮班族可自由勾选）。
  3. 设置 Work Start Time / Work End Time：可以从下拉里选 30 分钟一档的预设，也可以直接在框里键入任意时间（例如 8:45、17:30、9 PM）。
  4. 点击 Save and Apply。

🤖 它是怎么工作的？

启用后，程序每 5 分钟检查一次：

  • 当前在工作时间内：把 OOF 设为 Disabled（关闭自动回复）
  • 当前在工作时间外或非工作日：把 OOF 设为 Enabled（开启自动回复，使用你保存的内/外回复内容）

每次自动切换都会在主界面底部状态栏留下一条带时间戳的提示，例如 ✅ 18:00 · Outside working hours: OOF turned on automatically，方便你确认排程确实在跑。

💡 提示：点击 Check Now 可立即跑一次同步，方便你测试。
⚠️ 注意：排程依赖你电脑保持开机或休眠唤醒后联网。如果整周关机，OOF 状态会停留在最后一次同步的状态。

7使用模板库

在 📁 My Templates 卡片中可以保存常用文案：

  1. 在内/外回复编辑框写好内容。
  2. 点击 📋 Save as Template，给模板起个名字（例如"年假"、"出差"、"培训"）。
  3. 下次需要时，在模板列表点击对应条目右侧的 Load 即可填入回复框。
  4. 不再需要时，可以删除模板。
📂 模板存储位置： %LocalAppData%\OofManager\templates.db（SQLite，纯本地）

8系统托盘 / 关闭按钮行为

OOF Manager 支持最小化到系统托盘后台运行：

图 4 · 托盘右键菜单

托盘操作

  • 左键单击托盘图标 → 还原窗口
  • 右键 → 弹出菜单：Show Window / Hide to Tray / Exit / 重置关闭按钮行为

自定义关闭按钮行为

点击主窗口右上角的 ❌ 关闭按钮时，程序会询问你想要什么行为：

  • 📥 Hide to Tray — 最小化到托盘，程序继续运行
  • ❌ Exit — 完全退出（排程将停止）
  • Cancel — 不关闭

勾选 "Remember my choice, don't ask again" 即可记住选择。要重置该设置，从托盘菜单选 Reset Close Button Behavior (Ask Again Next Time)。

⚠️ 重要：如果选择 Exit 完全退出，自动排程也会停止。要让它持续生效，请使用 Hide to Tray。

9常见问题

Q：登录后报 "Connected, but failed to read the current signed-in user"

A：通常是网络中途断开或 Exchange Online 端临时拒绝。点击 Sign Out 后重试。如果持续失败，检查是否能访问 outlook.office365.com。

Q：保存提示 "Save request was sent but the server-side state was not updated…"

A：这表示请求已发出但服务器端没生效。可能原因：

  • 租户管理员的 OWA / Exchange 策略禁止用户修改自动回复
  • 登录账号的角色没有 Set-MailboxAutoReplyConfiguration 权限
  • 极少数情况下，ExchangeOnlineManagement 模块版本与租户端点不兼容

遇到这种情况请联系你的 IT / Exchange 管理员确认。

Q：SmartScreen 提示"Windows protected your PC"

A：因为安装包目前未购买代码签名证书。点击 More info → Run anyway。源码完全公开在 GitHub 可供审计。

Q：开了排程，第二天 OOF 没自动开

A：检查：

  • 程序是否仍在运行（任务栏 / 托盘是否能看到）
  • 登录会话是否还有效（Sign Out 后重新登录）
  • 电脑过夜是否一直处于联网状态（深度睡眠会让网络停止）
  • 用 Check Now 强制同步一次看是否成功

Q：日志在哪里？

A：当前版本主要通过界面状态栏反馈。后续版本会增加文件日志。

⌫卸载

  1. 打开 Windows 设置 → 应用 → 已安装的应用。
  2. 搜索 OOF Manager → 点击右侧 ··· → Uninstall。
  3. 卸载会移除程序本体，但保留你的偏好和模板（在 %LocalAppData%\OofManager）。
  4. 如果你想一并删除偏好与模板，运行下面这条 PowerShell 命令：
Remove-Item -Recurse -Force "$env:LocalAppData\OofManager"
